Privacy Policy

How Obiter handles your data. The short version: it mostly does not.

Last updated: 25 April 2026

Who We Are

Obiter is developed and maintained by Matthew Watt, an individual developer based in Australia. The project website is obiter.com.au. Obiter is a free, open-source Microsoft Word add-in that automates AGLC4 citation formatting.

Data Stored in Your Document

When you use Obiter to insert citations, the citation metadata is stored in a Custom XML Part within the .docx file itself. This data never leaves your document. It includes: source type, author names, titles, years, report series, pinpoints, and short titles.

All citation data remains under your full control. It is saved as part of your Word document and is not transmitted to any server during normal add-in operation.

Data Stored Locally

The following preferences are stored in your browser's localStorage on your device:

  • Template preferences
  • AGLC version selection
  • Branding preferences
  • LLM API keys (if configured)

No data stored in localStorage is transmitted to Obiter servers.

No Telemetry, No Analytics

Obiter does not collect usage data, analytics, crash reports, or any telemetry. No data is sent to obiter.com.au or any other server during normal add-in operation.

Third-Party LLM Integration (Optional, User-Controlled)

LLM features are entirely optional and disabled by default. Users must explicitly enable LLM integration and provide their own API key before any data is sent to a third-party provider.

When enabled, the following data may be sent to the user's chosen LLM provider (OpenAI, Anthropic, or a custom endpoint):

  • User-selected text only. Only text that you explicitly select and submit is sent. Obiter never sends your entire document, unseen content, or background data to any LLM.
  • Citation verification: the selected citation text and relevant AGLC4 rule text.
  • Citation parsing: a raw citation string pasted by you.
  • Short title suggestion: the citation's title and source type.
  • Source type classification: a user-provided description of a source.
  • Future features (e.g., "find me a reference") will always be limited to user-selected text that you explicitly review and approve before submission.

API keys are stored locally in your browser and are never written to your document or transmitted to the Obiter server. All LLM requests go directly from your browser to the provider you have selected (OpenAI, Anthropic, Google Gemini, xAI Grok, or DeepSeek). No request data, API keys, prompts, or responses pass through the Obiter server. Obiter has no access to your LLM provider account or usage data.

You can disable LLM integration at any time in Settings.

External API Search (Optional, Disabled by Default)

Obiter includes an optional typeahead search feature for cases and legislation. This feature is disabled by default and must be explicitly enabled in Settings. You can choose which search providers to use individually: AustLII, Jade.io, and the Federal Register of Legislation.

When enabled, search queries are routed through the Obiter proxy server (obiter.com.au) to bypass browser security restrictions, and are forwarded only to the providers you have selected. The proxy does not log, store, or retain search queries. Search is triggered only by your typing in a citation field — no automatic or background searches are performed.

Website Contact Forms and Signatures

Contact form submissions (name, email, message) are stored in a database on the Obiter server and forwarded via email to the developer.

Open letter signatures (name, title, institution, email) are stored in a database. Email addresses are used for verification only and are never displayed publicly. Signatures can be withdrawn at any time by contacting the developer.

Children

Obiter is not directed at children under 13. We do not knowingly collect personal information from children.

Changes to This Policy

We may update this privacy policy from time to time. Changes will be posted on this page with an updated "last updated" date.
